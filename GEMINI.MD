# 🚀 Project: Universal (Idle Clicker)

**Universal** — это idle-кликер о выживании одинокого корабля в недружелюбной галактике. Ваша цель: собирать мусор, расширять отсеки и накопить достаточно мощности для финального гиперпрыжка.

---

## 🛠 Технологический стек
*   **Engine:** Godot 4.x (GDScript)
*   **Architecture:** Event-Driven (Signaling Bus)
*   **AI Integration (Critical):** 
    *   **Context7 MCP:** Для получения актуальной документации по библиотекам Godot и лучшим практикам.
    *   **Godot-MCP Addon:** Для прямого управления сценами, нодами и скриптами через LLM.

---

## 🎯 Спецификация MVP (Хакатон "Чайник")

### 📏 Игровое поле и сетка
*   **Сетка:** 12 × 20 ячеек.
*   **Размер ячейки (Unit):** 90 × 90 пикселей.
*   **Стартовый корабль:** Зона 2 × 3 клетки (Ядро 2х1, Грузовой отсек 1х1, Сборщик 1х1).
*   **Ограничение:** Максимальный размер корабля для MVP — 6 × 6 ячеек.

### 💰 Экономика и Ресурсы
*   **Ресурс:** Только "Металл".
*   **Лимит металла:** `10 + (количество складов × 20)`.
*   **Стоимость модулей:**
    *   Расширение корпуса: 3 металла.
    *   Склад (+20 к лимиту): 5 металла.
    *   Сборщик (авто-сбор): 10 металла.

### ⚙️ Основные механики
1.  **Генерация мусора:** Мусор (1х1) спавнится справа, летит влево (1 клетка / 0.5 сек). Максимум 5 объектов на экране.
2.  **Сбор:** 
    *   Ручной: клик по мусору (+1 металл).
    *   Автоматический: Модуль "Сборщик" собирает мусор в радиусе 1 клетки (крестом).
3.  **Строительство:** Проходит в 2 этапа. Сначала покупка "Расширения корпуса" на соседней клетке, затем установка модуля.

### 🏆 Условие победы
*   Размер корабля 6 × 6.
*   Минимум 3 сборщика и 2 склада.
*   Полностью автоматизированный приток ресурсов.

## 📱 Интерфейс (UI/UX)
*   **Ориентация:** Вертикальная (1080 × 2400).
*   **SafeArea:** Отступы 150–200 пикселей сверху и снизу.
*   **Панели:** 
    *   Верх (2 строки сетки): Металл/Лимит, размер корабля.
    *   Низ (2 строки сетки): Кнопки магазина (серые, если нет денег).
*   **Камера:** Динамический зум. Если ширина корабля > 10 ячеек, масштаб уменьшается до 60px/ячейка.

## 🏗 Архитектурные принципы
1.  **Feature-based Folders:** Все ресурсы фичи в одной папке.
2.  **Event Bus:** Глобальный синглтон `GameEvents`. Никаких `get_node("../../../")`.
3.  **Composition over Inheritance:** Собираем функционал из компонентов.

## 📂 Структура проекта
```text
res://
├── addons/                 # Плагины, включая godot_mcp
├── assets/                 # Общие ассеты (звуки, шрифты, палитры)

│   ├── game_events.gd      # "Почта" для сигналов между модулями
│   └── resource_manager.gd # Хранение Металла и Энергии
├── data/                   # Файлы баланса (.tres)
├── entities/               # Игровые объекты
│   ├── ship/               # Сетка корабля и логика постройки
│   ├── modules/            # Сцены отсеков
│   └── debris/             # Космический мусор
├── ui/                     # HUD, Магазин, Меню
├── shared/                 # Общие компоненты (Health, Clickable)
└── scripts/                # Вспомогательная логика
```

## 🤝 Правила работы в Git (Workflow)

## 🧪 Тестирование

В проекте используется система модульных тестов на базе `EditorScript`. Все тесты находятся в папке `tests/`.

### Как запускать тесты:
1.  Откройте нужный файл теста (например, `tests/test_resource_manager.gd`) в редакторе Godot.
2.  Нажмите **File -> Run** (или `Ctrl + Shift + X` / `Cmd + Shift + X`).
3.  Результаты появятся в консоли **Output** редактора Godot.

### Что покрыто тестами:
*   **ResourceManager:** Логика сбора металла, лимитов и трат.
*   **ShipGrid:** Логика расширения корабля (соседство, максимальный размер) и расчет вместимости складов.
